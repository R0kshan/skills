## Task
Complete the code coverage by writting new tests or completing existing tests in $ARGUMENTS.

## Steps

1. Read $ARGUMENTS and the class it is testing
2. Verify that the test class covers 90% of the code by checking the report generated by the /junit-verify-code-coverage skill
3. If line, method or branch coverage is less than 90% or the code is not compilable or tests fail, complete the existing test class to ensure code coverage and repeat step 2. Else stop here.

## Hard rules
- Only write code to $ARGUMENTSTest class
- All public and protected methods must be tested
- Private methods should be tested only through calls from public methods
- All branches, lines and statements must be covered 
- The output of the tested methods must ALWAYS BE VERIFIED with assertions
- The tests must be COMPILABLE
- All tests must PASS
- Use Mocking and Stubs (using classes available from maven dependencies)
- Keep Tests Small and Focused - each test should verify a single behavior or functionality and avoid testing multiple scenarios in one test
- Ensure Tests are Deterministic - tests should produce the same result every time they run and do not rely on external factors like network calls or system time
- Use descriptive test names
- Test Edge Cases and Error Conditions
- Follow the AAA Pattern (Arrange, Act, Assert)
    - Arrange: Set up the test environment and inputs.
    - Act: Execute the function or method being tested.
    - Assert: Verify the expected outcome.
- Code and comments must be written in FRENCH
- The modifications mustn't introduce Sonar warnings
